## Mandatory language and currency rules

This project must now follow these product rules everywhere:

* **French is the default language across the entire platform**
* **Arabic is the secondary language that users can switch to**
* the app must open and operate in **French by default**
* the public website, customer ordering flow, merchant dashboard, admin panel, billing pages, onboarding, and operational screens must all be **French-first**
* Arabic must be supported as an optional switchable language
* if Arabic translation is missing for a field or UI section, fallback to French
* all prices and financial values must be displayed in **Algerian dinar**
* use **DA** as the main visible currency format in the UI
* store monetary values safely in the database and format them consistently in the frontend as examples like:

  * `250 DA`
  * `1 200 DA`
  * `3 500 DA`

The multilingual structure must safely support:

* **French (default / LTR)**
* **Arabic (secondary / RTL)**

All UI components must remain stable in both French and Arabic.
Do not make Arabic the default language.
Do not build English as the main product language.
French-first is mandatory.
